Use this endpoint to confirm that your session is active and to retrieve your Pay3 user context — including your internal user ID, wallet ID, Stellar wallet address, network, and session expiry. This is especially useful after page load or app initialization to check whether a stored session cookie is still valid before making further API calls.

## Endpoint

**`GET /api/auth/me`**

Authentication is **required**. Include either the `pay3_session` cookie or an `Authorization: Bearer <session-token>` header.

---

## Response

All Pay3 API responses are wrapped in a standard envelope:

```json
{
  "data": { ... },
  "requestId": "req_01hx2q3r4s5t6u7v8w9x0y1z"
}
```

A successful `200 OK` response returns your authenticated user context inside `data`.

<ResponseField name="data" type="object">
  The authenticated user context.

  <Expandable title="data fields" defaultOpen>
    <ResponseField name="data.userId" type="string">
      Your Pay3 internal user ID. Stable across sessions and wallet
      re-connections.
    </ResponseField>

    <ResponseField name="data.walletId" type="string">
      The Pay3 internal record ID for your currently authenticated Stellar
      wallet.
    </ResponseField>

    <ResponseField name="data.walletAddress" type="string">
      Your Stellar public key (`G...`) associated with the active session.
    </ResponseField>

    <ResponseField name="data.network" type="string">
      The Stellar network for this session — `testnet` or `public`.
    </ResponseField>

    <ResponseField name="data.sessionExpiresAt" type="string">
      ISO 8601 timestamp indicating when the current session expires. Use
      this to proactively re-authenticate before the session ends.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="requestId" type="string">
  A unique identifier for this API request, useful for support and debugging.
</ResponseField>

## Example

### Request

```bash
# Using a Bearer token
curl --request GET \
  --url https://api.pay3.dev/api/auth/me \
  --header 'Authorization: Bearer your-session-token-here'
```

```bash
# Using the session cookie (browser)
curl --request GET \
  --url https://api.pay3.dev/api/auth/me \
  --cookie 'pay3_session=your-session-token-here'
```

### Response Body

```json
{
  "data": {
    "userId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "walletId": "f0e1d2c3-b4a5-6789-0123-456789abcdef",
    "walletAddress": "GAHJJJKMOKYE4RVPZEWZTKH5FVI4PA3VL7GK2LFNUBSGBV8XVMHXV6R",
    "network": "testnet",
    "sessionExpiresAt": "2024-11-08T12:00:00.000Z"
  },
  "requestId": "req_01hx2q3r4s5t6u7v8w9x0y1z"
}
```

## Error Responses

| HTTP Status | Description |
|---|---|
| `401 Unauthorized` | No session token was provided, or the token has expired or been revoked. Re-authenticate via [`POST /api/auth/challenge`](/api-reference/auth/challenge) and [`POST /api/auth/verify`](/api-reference/auth/verify). |
