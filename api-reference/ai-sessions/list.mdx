Retrieve every currently active AI session associated with your account. Each session represents an authorized AI client — such as Cursor, Claude, or ChatGPT — that has been granted permission to initiate Stellar transactions on your behalf within the bounds of a spending policy. Use this endpoint to audit which AI assistants are active and to identify sessions that should be revoked.

## Endpoint

```
GET /api/ai-sessions
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

## Response

Returns an array of active AI session objects, ordered newest first. Only sessions with `ACTIVE` status and a future expiry time are returned — expired and revoked sessions are excluded. To revoke a session, see [Revoke Sessions](/api-reference/ai-sessions/revoke).

### Response fields

<ResponseField name="id" type="string" required>
  Unique UUID identifying this AI session.
</ResponseField>

<ResponseField name="clientType" type="string" required>
  The AI client associated with this session. One of `CURSOR`, `CLAUDE`, `CHATGPT`, `GEMINI`, or `OTHER`.
</ResponseField>

<ResponseField name="sessionPublicKey" type="string" required>
  The Ed25519 public key (Stellar G-address) of the ephemeral session keypair registered on-chain. This key signs transactions submitted by the AI on your behalf.
</ResponseField>

<ResponseField name="expiresAt" type="string" required>
  ISO 8601 timestamp after which the session automatically becomes inactive.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the session was created.
</ResponseField>

<ResponseField name="policy" type="object" required>
  Summary of the spending policy governing this session's transaction limits.

  <Expandable title="policy fields">
    <ResponseField name="id" type="string" required>
      UUID of the spending policy.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Display name of the policy.
    </ResponseField>
    <ResponseField name="preset" type="string" required>
      Policy preset. One of `CONSERVATIVE`, `BALANCED`, `DEFI`, or `CUSTOM`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="smartAccount" type="object" required>
  The smart account (Soroban contract) this session is authorized to operate on.

  <Expandable title="smartAccount fields">
    <ResponseField name="id" type="string" required>
      UUID of the smart account record.
    </ResponseField>
    <ResponseField name="contractId" type="string" required>
      On-chain Soroban contract ID (C-address).
    </ResponseField>
  </Expandable>
</ResponseField>

## Example request

```bash
curl https://api.pay3.dev/api/ai-sessions \
  -H "Authorization: Bearer <token>"
```

</Note>
