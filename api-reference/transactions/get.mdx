Retrieve the complete record for a single AI-initiated transaction. In addition to the fields returned by the list endpoint, the detail view includes the embedded approval request (if one exists), failure reasons, and precise timestamps for each stage of the submission pipeline. Use this endpoint when investigating a specific transaction â€” for example, to diagnose a failure, confirm on-chain settlement, or understand why a transaction was held for approval.

## Endpoint

```
GET /api/transactions/:id
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

## Path parameters

<ParamField path="id" type="string" required>
  The UUID of the transaction to retrieve.
</ParamField>

## Response

Returns a single transaction object with all available fields. Returns `404 Not Found` if the transaction does not exist or does not belong to your account.

### Response fields

<ResponseField name="id" type="string" required>
  Unique UUID for this transaction.
</ResponseField>

<ResponseField name="action" type="string" required>
  The operation type. One of `TRANSFER`, `CHECK_BALANCE`, or `VIEW_TRANSACTION_HISTORY`. See [GET /api/transactions](/api-reference/transactions/list) for full descriptions.
</ResponseField>

<ResponseField name="asset" type="string" required>
  Ticker of the asset involved, e.g. `"USDC"` or `"XLM"`.
</ResponseField>

<ResponseField name="amount" type="string">
  Transfer amount as a decimal string. May be `null` for non-transfer actions.
</ResponseField>

<ResponseField name="recipientAddress" type="string">
  Resolved destination Stellar address (G-address). `null` for non-transfer actions.
</ResponseField>

<ResponseField name="recipientInput" type="string">
  The raw recipient value originally provided by the AI â€” may be an alias, display name, or Stellar address.
</ResponseField>

<ResponseField name="status" type="string" required>
  Lifecycle status. One of `CREATED`, `VALIDATING`, `AUTO_APPROVED`, `PENDING_APPROVAL`, `APPROVED`, `SIGNING`, `SUBMITTING`, `SUCCESS`, `FAILED`, `REJECTED`, `EXPIRED`, or `CANCELLED`. See [GET /api/transactions](/api-reference/transactions/list) for full status descriptions.
</ResponseField>

<ResponseField name="policyDecision" type="string" required>
  The decision reached by the policy engine when the transaction was first evaluated:
  - `AUTO_EXECUTE` â€” the transaction was within all limits and was submitted without requiring your approval
  - `PENDING_APPROVAL` â€” the transaction exceeded the `manualApprovalThreshold` and was routed to the approval queue
  - `REJECTED` â€” the transaction violated a policy rule and was blocked before submission
</ResponseField>

<ResponseField name="stellarTransactionHash" type="string">
  The Stellar transaction hash. Populated once status is `SUCCESS`. `null` for all other statuses.
</ResponseField>

<ResponseField name="rejectionReason" type="string">
  Human-readable explanation of why the transaction failed or was rejected. Populated when `status` is `FAILED` or `REJECTED`. `null` for all other statuses.
</ResponseField>

<ResponseField name="aiSessionId" type="string" required>
  UUID of the AI session that submitted this transaction.
</ResponseField>

<ResponseField name="aiSession" type="object" required>
  Summary of the AI session that submitted this transaction.

  <Expandable title="aiSession fields">
    <ResponseField name="id" type="string" required>
      UUID of the AI session.
    </ResponseField>
    <ResponseField name="clientType" type="string" required>
      AI client type (e.g. `"CURSOR"`, `"CLAUDE"`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="approvalRequest" type="object">
  The approval request record associated with this transaction, if one was created (i.e. `policyDecision` is `PENDING_APPROVAL`). `null` if the transaction was auto-executed or rejected before reaching the approval stage.

  <Expandable title="approvalRequest fields">
    <ResponseField name="id" type="string" required>
      UUID of the approval request.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Approval request status. One of `PENDING`, `APPROVED`, `EXPIRED`, or `CANCELLED`.
    </ResponseField>
    <ResponseField name="expiresAt" type="string" required>
      ISO 8601 timestamp after which the approval request expires.
    </ResponseField>
    <ResponseField name="approvedAt" type="string">
      ISO 8601 timestamp when you approved the request. `null` if not yet approved.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp when the approval request was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the transaction record was created.
</ResponseField>

<ResponseField name="completedAt" type="string">
  ISO 8601 timestamp when the transaction reached a terminal state (`SUCCESS`, `FAILED`, `REJECTED`, `EXPIRED`, or `CANCELLED`). `null` if still in progress.
</ResponseField>

## Error responses

| Status | Meaning |
|--------|---------|
| `400 Bad Request` | The provided `id` is not a valid UUID. |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | No transaction exists for the given `id` on your account. |
