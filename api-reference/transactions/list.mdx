Retrieve a list of every transaction submitted by your AI sessions, ordered newest first. Each record includes the asset, amount, recipient Stellar address, current lifecycle status, and ‚Äî once confirmed ‚Äî the Stellar transaction hash you can look up on any Stellar explorer. Use this endpoint to audit spending, reconcile balances, and track which transactions are still in flight.

## Endpoint

```
GET /api/transactions
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

## Query parameters

<ParamField query="limit" type="number" default="50">
  Maximum number of transactions to return. Defaults to `50`. Pass a lower value to reduce response size.
</ParamField>

## Response

Returns an array of transaction objects ordered by `createdAt` descending (newest first).

### Response fields

<ResponseField name="id" type="string" required>
  Unique UUID for this transaction.
</ResponseField>

<ResponseField name="action" type="string" required>
  The operation type. One of:
  - `TRANSFER` ‚Äî an asset transfer to a recipient address
  - `CHECK_BALANCE` ‚Äî a balance read (does not move funds)
  - `VIEW_TRANSACTION_HISTORY` ‚Äî a history read (does not move funds)
</ResponseField>

<ResponseField name="asset" type="string" required>
  Ticker of the asset involved, e.g. `"USDC"` or `"XLM"`.
</ResponseField>

<ResponseField name="amount" type="string">
  Amount involved, expressed as a decimal string (e.g. `"42.5000000"`). May be `null` for non-transfer actions.
</ResponseField>

<ResponseField name="recipientAddress" type="string">
  The resolved destination Stellar address (G-address). Populated for `TRANSFER` actions once the recipient has been resolved. `null` for non-transfer actions.
</ResponseField>

<ResponseField name="recipientInput" type="string">
  The raw recipient value supplied by the AI, which may be a Stellar address, a contact alias, or a display name. Use this to understand what the AI asked to send to.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current lifecycle status of the transaction. One of:
  - `CREATED` ‚Äî transaction record created, not yet validated
  - `VALIDATING` ‚Äî policy engine is evaluating the request
  - `AUTO_APPROVED` ‚Äî passed all policy checks; being submitted automatically
  - `PENDING_APPROVAL` ‚Äî exceeds `manualApprovalThreshold`; awaiting your review
  - `APPROVED` ‚Äî you approved it; being submitted to the network
  - `SIGNING` ‚Äî transaction envelope is being signed
  - `SUBMITTING` ‚Äî signed envelope is being broadcast to the Stellar network
  - `SUCCESS` ‚Äî confirmed on the Stellar network
  - `FAILED` ‚Äî submission failed; see `rejectionReason` via [GET /api/transactions/:id](/api-reference/transactions/get)
  - `REJECTED` ‚Äî rejected by the policy engine before submission
  - `EXPIRED` ‚Äî the approval window elapsed before you approved
  - `CANCELLED` ‚Äî cancelled before reaching the network
</ResponseField>

<ResponseField name="policyDecision" type="string" required>
  The decision made by the policy engine:
  - `AUTO_EXECUTE` ‚Äî transaction was within all limits and submitted without approval
  - `PENDING_APPROVAL` ‚Äî transaction exceeded `manualApprovalThreshold` and was routed to the approval queue
  - `REJECTED` ‚Äî transaction violated a policy rule and was blocked outright
</ResponseField>

<ResponseField name="stellarTransactionHash" type="string">
  The Stellar transaction hash, populated once the transaction reaches `SUCCESS`. Use this to look up the transaction on [Stellar Expert](https://stellar.expert) or [Horizon](https://horizon.stellar.org).
</ResponseField>

<ResponseField name="aiSessionId" type="string" required>
  UUID of the AI session that initiated this transaction.
</ResponseField>

<ResponseField name="aiSession" type="object" required>
  Summary of the AI session that submitted this transaction.

  <Expandable title="aiSession fields">
    <ResponseField name="id" type="string" required>
      UUID of the AI session.
    </ResponseField>
    <ResponseField name="clientType" type="string" required>
      AI client type (e.g. `"CURSOR"`, `"CLAUDE"`).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="contact" type="object">
  If the recipient address matched a saved contact, this object contains the contact's details. `null` when no match was found.

  <Expandable title="contact fields">
    <ResponseField name="id" type="string" required>
      UUID of the matched contact.
    </ResponseField>
    <ResponseField name="displayName" type="string" required>
      Human-readable name of the contact.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the transaction was created.
</ResponseField>

## Example request

```bash
curl "https://api.pay3.dev/api/transactions?limit=20" \
  -H "Authorization: Bearer <token>"
```
