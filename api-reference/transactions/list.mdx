Retrieve a paginated list of every transaction submitted by your AI sessions. Each record includes the asset, amount, recipient Stellar address, current lifecycle status, and — once confirmed — the Stellar transaction hash you can look up on any Stellar explorer. Use this endpoint to audit spending, reconcile balances, and track which transactions are still in flight.

## Endpoint

```
GET /api/transactions
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

## Query parameters

<ParamField query="limit" type="number" default="20">
  Maximum number of transactions to return per page. Defaults to `20`.
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip before starting the result set. Use with `limit` to paginate through large histories.
</ParamField>

## Response

Returns an array of transaction objects ordered by `createdAt` descending (newest first).

### Response fields

<ResponseField name="id" type="string" required>
  Unique UUID for this transaction.
</ResponseField>

<ResponseField name="action" type="string" required>
  The operation type. Currently always `TRANSFER` for spend transactions.
</ResponseField>

<ResponseField name="asset" type="string" required>
  Ticker of the asset being sent, e.g. `"USDC"` or `"XLM"`.
</ResponseField>

<ResponseField name="amount" type="string" required>
  Amount being transferred, expressed as a decimal string (e.g. `"42.5000000"`).
</ResponseField>

<ResponseField name="recipient" type="string" required>
  The destination Stellar address (G-address) for this transaction.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current lifecycle status of the transaction. One of:
  - `PENDING` — waiting in the approval queue (requires your manual approval)
  - `SIGNING` — Pay3 is signing the Stellar transaction envelope
  - `SUBMITTING` — the signed envelope is being broadcast to the Stellar network
  - `SUCCESS` — transaction confirmed on Stellar
  - `FAILED` — transaction failed; see the full detail endpoint for `rejectionReason`
  - `EXPIRED` — the approval window closed before you approved it
</ResponseField>

<ResponseField name="stellarHash" type="string">
  The Stellar transaction hash, populated once the transaction reaches `SUCCESS`. Use this to look up the transaction on [Stellar Expert](https://stellar.expert) or [Horizon](https://horizon.stellar.org).
</ResponseField>

<ResponseField name="sessionId" type="string" required>
  UUID of the AI session that initiated this transaction.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the transaction was created.
</ResponseField>

## Example request

```bash
curl "https://api.pay3.dev/api/transactions?limit=20&offset=0" \
  -H "Authorization: Bearer <token>"
```
