Retrieve every transaction currently waiting for your manual approval. When an AI session attempts a transfer that exceeds the `manualApprovalThreshold` set in the session's spending policy, Pay3 holds the transaction and creates an approval request instead of executing it immediately. You must review and approve (or simply let it expire) within 2 minutes. This endpoint gives you the full list of open approval requests so you can process them promptly.

## Endpoint

```
GET /api/approvals/pending
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

<Warning>
  Each approval request expires **2 minutes** after it is created. If you do not approve the transaction before `expiresAt`, the request is automatically cancelled and the underlying transaction moves to `EXPIRED` status. The AI session will need to retry the operation if the action is still desired.
</Warning>

## Response

Returns an array of open approval request objects, ordered by `createdAt` descending (most recently created first). Only requests with a future `expiresAt` are included — already-expired requests do not appear.

### Response fields

<ResponseField name="id" type="string" required>
  UUID of the approval request itself.
</ResponseField>

<ResponseField name="status" type="string" required>
  Always `PENDING` for entries returned by this endpoint.
</ResponseField>

<ResponseField name="transactionId" type="string" required>
  UUID of the transaction awaiting approval. Pass this to [POST /api/approvals/:txId/approve](/api-reference/approvals/approve) to execute it.
</ResponseField>

<ResponseField name="aiSessionId" type="string" required>
  UUID of the AI session that submitted the transaction request.
</ResponseField>

<ResponseField name="expiresAt" type="string" required>
  ISO 8601 timestamp 2 minutes after `createdAt`. The approval request is cancelled automatically if you have not approved it by this time.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when the approval request was created (i.e. when the AI submitted the transaction).
</ResponseField>

<ResponseField name="transaction" type="object" required>
  The full transaction record for the payment being held for review.

  <Expandable title="transaction fields">
    <ResponseField name="id" type="string" required>
      UUID of the transaction.
    </ResponseField>
    <ResponseField name="action" type="string" required>
      Always `TRANSFER` for transactions that require approval.
    </ResponseField>
    <ResponseField name="asset" type="string" required>
      Asset code to be transferred (e.g. `"USDC"`).
    </ResponseField>
    <ResponseField name="amount" type="string" required>
      Requested transfer amount as a decimal string.
    </ResponseField>
    <ResponseField name="recipientAddress" type="string" required>
      Resolved Stellar G-address that will receive the funds if you approve.
    </ResponseField>
    <ResponseField name="recipientInput" type="string" required>
      The raw recipient value the AI originally specified.
    </ResponseField>
    <ResponseField name="status" type="string" required>
      Transaction status — `PENDING_APPROVAL` for all entries in this context.
    </ResponseField>
    <ResponseField name="policyDecision" type="string" required>
      Always `PENDING_APPROVAL` here.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp when the transaction record was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="aiSession" type="object" required>
  Summary of the AI session that triggered this approval request.

  <Expandable title="aiSession fields">
    <ResponseField name="id" type="string" required>
      UUID of the AI session.
    </ResponseField>
    <ResponseField name="clientType" type="string" required>
      AI client that initiated the transaction (e.g. `"CURSOR"`, `"CLAUDE"`).
    </ResponseField>
  </Expandable>
</ResponseField>

## Example request

```bash
curl https://api.pay3.dev/api/approvals/pending \
  -H "Authorization: Bearer <token>"
```
