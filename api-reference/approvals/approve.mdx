Approve a transaction that is waiting in the manual review queue. Once you call this endpoint, Pay3 immediately signs the Stellar transaction envelope and broadcasts it to the network — no further action is needed on your part. The response is the updated transaction record, which will include the Stellar transaction hash once the network confirms the payment. Approval requests expire 2 minutes after they are created, so act promptly after retrieving the pending list.

## Endpoint

```
POST /api/approvals/:txId/approve
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

## Path parameters

<ParamField path="txId" type="string" required>
  The UUID of the **transaction** to approve. Obtain this from the `transactionId` field in [GET /api/approvals/pending](/api-reference/approvals/list).
</ParamField>

## Request body

No request body is required.

## Response

Returns the updated transaction record after submission. The shape mirrors the objects returned by [GET /api/transactions/:id](/api-reference/transactions/get).

<ResponseField name="id" type="string" required>
  UUID of the transaction.
</ResponseField>

<ResponseField name="status" type="string" required>
  Updated transaction status. Will be `SUBMITTING` immediately after approval while the Stellar network processes the payment, and will eventually transition to `SUCCESS` or `FAILED`.
</ResponseField>

<ResponseField name="stellarTransactionHash" type="string">
  The Stellar transaction hash. Populated once the transaction reaches `SUCCESS`. May be `null` if the network confirmation has not yet been received at the time the response is returned — poll [GET /api/transactions/:id](/api-reference/transactions/get) to retrieve the final hash.
</ResponseField>

<ResponseField name="action" type="string" required>
  Always `TRANSFER` for transactions that pass through the approval queue.
</ResponseField>

<ResponseField name="asset" type="string" required>
  Asset code transferred.
</ResponseField>

<ResponseField name="amount" type="string" required>
  Transfer amount as a decimal string.
</ResponseField>

<ResponseField name="recipientAddress" type="string" required>
  Destination Stellar G-address.
</ResponseField>

<ResponseField name="policyDecision" type="string" required>
  Always `PENDING_APPROVAL` for transactions that went through the approval queue.
</ResponseField>

<ResponseField name="completedAt" type="string">
  ISO 8601 timestamp when the transaction reached `SUCCESS`. `null` if network confirmation is still pending.
</ResponseField>

## Error responses

| Status | Meaning |
|--------|---------|
| `400 Bad Request` | The provided `txId` is not a valid UUID. |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | No pending approval request exists for the given `txId`, it does not belong to your account, or it has already been acted upon. |
| `410 Gone` | The approval request has expired (the 2-minute window elapsed before you approved). The transaction status is now `EXPIRED`. |

<Warning>
  **Approval is irreversible.** The moment you call this endpoint the transaction is signed and submitted to the Stellar network. Funds will leave your smart account immediately. There is no undo — Stellar transactions are final. Verify the `amount`, `asset`, and `recipientAddress` in the [pending approvals list](/api-reference/approvals/list) before approving.
</Warning>

## Example request

```bash
curl -X POST https://api.pay3.dev/api/approvals/c3b2a1f0-9876-5432-fedc-ba0987654321/approve \
  -H "Authorization: Bearer <token>"
```

## Example error response — expired approval

```json
{
  "error": "Gone",
  "message": "This approval request has expired and can no longer be acted on."
}
```

<Note>
  If `stellarTransactionHash` is `null` in the response, the Stellar network confirmation is still pending. Retrieve the final state — including the hash once confirmed — via [GET /api/transactions/:id](/api-reference/transactions/get).
</Note>
