The dashboard endpoint is your single entry point for building a complete account overview. One authenticated request returns seven data points in parallel â€” wallet balances, recent transactions, active AI sessions, pending approval requests, spending policies, monthly usage records, and per-session daily budget consumption. Rather than making seven separate API calls and stitching the results together on the client, you can use this endpoint to populate an entire dashboard UI in a single round trip.

<Tip>
  This endpoint is purpose-built for dashboard UIs. Because the server fetches all seven data sources concurrently with `Promise.all`, the total response time is bounded by the slowest individual source â€” not the sum of all seven. Use it wherever you need a full account snapshot and want to minimize network overhead.
</Tip>

## Endpoint

```
GET /api/dashboard
```

## Authentication

All requests must include a valid bearer token in the `Authorization` header, or a `pay3_session` cookie obtained during wallet authentication.

```
Authorization: Bearer <token>
```

  All active spending policies on your account, ordered newest first.

  <Expandable title="policies[] fields">
    <ResponseField name="id" type="string" required>
      UUID of the policy.
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Human-readable name you gave this policy.
    </ResponseField>
    <ResponseField name="preset" type="string" required>
      Policy preset used as the starting configuration. One of `CONSERVATIVE`, `BALANCED`, `DEFI`, or `CUSTOM`.
    </ResponseField>
    <ResponseField name="dailyBudget" type="string" required>
      Maximum total spend per day across all sessions using this policy, as a decimal string (USD).
    </ResponseField>
    <ResponseField name="perTransactionMax" type="string" required>
      Maximum amount for a single transaction, as a decimal string (USD).
    </ResponseField>
    <ResponseField name="manualApprovalThreshold" type="string" required>
      Transactions above this amount (USD) require manual approval before submission.
    </ResponseField>
    <ResponseField name="sessionDurationHours" type="number" required>
      Maximum lifetime in hours for any session created under this policy.
    </ResponseField>
    <ResponseField name="allowedAssets" type="array" required>
      List of asset codes this policy permits (e.g. `["USDC", "XLM"]`).
    </ResponseField>
    <ResponseField name="allowedActions" type="array" required>
      List of actions permitted under this policy. Values include `CHECK_BALANCE`, `TRANSFER`, and `VIEW_TRANSACTION_HISTORY`.
    </ResponseField>
    <ResponseField name="allowedRecipientIds" type="array" required>
      List of contact UUIDs that sessions under this policy may send to. An empty array means no recipient restriction is enforced.
    </ResponseField>
    <ResponseField name="monthlyBudget" type="string">
      Optional monthly spend cap in USD. `null` if no monthly limit is set.
    </ResponseField>
    <ResponseField name="isActive" type="boolean" required>
      Whether this policy is currently active. Only active policies appear in this array.
    </ResponseField>
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp when the policy was created.
    </ResponseField>
  </Expandable>
</ResponseField>

    </ResponseField>

        <ResponseField name="asset" type="string" required>
          Asset code for this usage record (e.g. `"USDC"`).
        </ResponseField>
        <ResponseField name="totalAmount" type="string" required>
          Total amount transacted in this asset during the period, as a decimal string.
        </ResponseField>
        <ResponseField name="transactionCount" type="number" required>

        </ResponseField>
      </Expandable>

      </Expandable>
    </ResponseField>

        <ResponseField name="monthlySpent" type="array" required>

          <Expandable title="monthlySpent[] fields">

            <ResponseField name="spentAmount" type="string" required>

            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>
