## Why Pay3?

Today's payment systems were designed for humans. Credit cards, static API keys, and monthly SaaS subscriptions don't scale to autonomous software agents that need to make thousands of micro-transactions per day.

Pay3 solves this with a machine-native payment layer:

- **Your AI agent gets its own wallet** — a Stellar keypair with delegated spending permissions, never the real private key.
- **You set the rules** — daily budgets, per-transaction limits, allowed assets, and approved recipient lists.
- **The policy engine enforces them** — every transaction is evaluated before it executes; out-of-policy requests are automatically rejected or queued for your approval.
- **Everything settles on Stellar** — fast (3–5 second finality), cheap, and USDC-native.

## Core Features

| Feature | Description |
|---|---|
| **AI Wallet** | Stellar-based wallet with delegated session keys; your private key never leaves your device |
| **Policy Engine** | Per-session daily/monthly budgets, per-tx limits, asset allowlists, and recipient restrictions |
| **MCP Server** | Three MCP tools (`get_balance`, `transfer`, `get_transaction_history`) compatible with any MCP client |
| **x402 Payments** | HTTP 402-based stateless micropayments for machine-to-machine API access |
| **Soroban Smart Account** | On-chain USDC vault enforcing limits at the contract level, independent of off-chain policy |
| **Manual Approvals** | Human-in-the-loop approval queue for high-value transactions |
| **DeFi Integrations** | Blend lending, Phoenix swaps, and Aquarius liquidity — coming in Phase 2 |

## Get Started

<Steps>
  <Step title="Connect your Stellar wallet">
    Install the Freighter browser extension and sign in at the Pay3 Dashboard using your Stellar wallet address.
  </Step>
  <Step title="Link a Smart Account">
    Deploy (or link an existing) Soroban smart account contract that acts as your AI's USDC vault.
  </Step>
  <Step title="Create a Spending Policy">
    Define daily/monthly budgets, per-transaction maximums, and which assets your AI is allowed to spend.
  </Step>
  <Step title="Create an AI Session">
    Generate an MCP auth token for your AI assistant and configure it in Claude Desktop, Cursor, or your agent framework.
  </Step>
</Steps>
